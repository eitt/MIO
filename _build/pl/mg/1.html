Método Gráfico

Introducción

El método gráfico para la resolución de problemas de programación lineal es generalmente utilizado de manera introductoria. Principalmente debido a que éste se aplica cuando existen sólo dos variables de decisión, condición que no es común en la mayoría de problemas; sin embargo, es a partir del análisis gráfico que es posible describir el Método Simplex que permite la resolución de problemas con más de dos variables.

En esencia, el método gráfico es un análisis del espacio de búsqueda y su correspondiente función objetivo, con la ventaja de poder representar de manera simultánea tres dimensiones que corresponden a los ejes de las abscisas ($x_1,x_2$) y el eje ordenado de la función objetivo, comúnmente descrito con la variable $z$. Por consiguiente, el método gráfico es el análisis de crecimiento o decrecimiento del plano $z$ que depende de las variables $x_1,x_2$, es decir, $z=f(x_1,x_2)$, donde esta relación se puede reescribir de la siguiente manera: $z-a_1x_1-a_2x_2=0$, que la respectiva ecuación de un plano, donde $a_1 y a_2$ son los coeficientes que pre multiplican a las variables de decisión.

De manera general, los problemas de programación lineal implican variables de decisión que compiten entre sí por el uso de recursos limitados, por ejemplo, para la construcción de implantes impresos en 3D, una compañía se especializa en el desarrollo de dos tipos de implantes:

  1. Implantes dentales.
  2. Implantes óseos.

Con el fin de simplificar, se puede suponer que sólo se fabrica una referencia de cada tipo de implante. Como ambos implantes utilizan la misma materia prima, es necesario decidir cuánto de cada tipo de implante se va a fabricar considerando elementos como la demanda de los productos, los recursos en común (maquinaria, empleados, materia prima, etc.), ingresos percibidos, y demás elementos o consideraciones que permitan describir el problema a resolver.

Esta situación (diversos productos que compiten entre sí por recursos) se denomina frecuentemente mezcla de productos, y su estructura es ampliamente utilizada en diversos campos como: construcción, agricultura, metalurgia, servicios, entre otros. {% cite Heady1954%}

Espacio de búsqueda en el método gráfico

Como se indica en los cinco pasos para modelar un PL, una vez definidas las variables, la función objetivo con su respectivo criterio, es necesario construir el espacio de búsqueda, o espacio de restricciones. Para ello, se puede partir de la siguiente inecuación: $𝑓(𝑥_1,𝑥_2)≤𝑏$ o $𝑓(𝑥_1,𝑥_2)≥𝑏$.

Se puede demostrar que, en dos dimensiones, el conjunto de puntos que satisface una desigualdad lineal (o inecuación) comprende o contiene los puntos en la recta $𝑓(𝑥_1,𝑥_2)=𝑏$, que define la desigualdad más todos los puntos de un lado de la recta (puntos por encima en el caso de ser mayor o igual $"≥"$, o los puntos inferiores para el escenario de menor igual $"≤"$). Sin embargo, en algunos casos no es tan intuitivo determinar qué lado de la recta corresponde a la desigualdad, por tanto, se elige un punto $𝑃$ arbitrario y a partir de este se evalúa la condición de la restricción.

Restricciones de igualdad

Normalmente un PL de dos variables, cuyas variables de decisión corresponden a $𝑥_1$ y $𝑥_2$, presenta una forma similar a la siguiente restricción:

\begin{equation*} -2𝑥_1+1.5𝑥_2=6 \end{equation*}

Donde los valores de $-2$ y de $1.5$ que pre multiplican a las variables $𝑥_1,𝑥_2$ son un ejemplo de las posibles realizaciones de la carga $c\in R$. Al observar la ecuación que describe la restricción se encuentra que el mismo conjunto de puntos satisface $(𝑥_1,𝑥_2)$ satisface:

\begin{equation*} 1.5𝑥_2=6+2𝑥_1 \end{equation*}

De igual forma satisface:

\begin{equation*} 𝑥_2=4−\frac{2}{1.5}𝑥_1=>𝑥_2=4−\frac{4}{3}𝑥_1 \end{equation*}

o:

\begin{equation*} 𝑥_1=\frac{1.5𝑥_2-6}{-2}=>𝑥_1=\frac{-3}{4}𝑥_2+3 \end{equation*}

Según la variable que se desee expresar como dependiente.

Ahora bien, esta relación al estar en $R^2$ puede ser representada de manera gráfica. En R es posible realizar gráficas de funciones, para ello son necesarios los siguientes pasos:

  1. Definir la función.
  2. Determinar el dominio de la variable en el eje de las abscisas.
  3. Utilizar un paquete package de gráficos.

Por lo general, como buena práctica se cargan los paquetes al principio de cada código. Como tal se trabajará con el paquete ggplot2 éste permite generar de manera flexible diferentes tipos de gráficos, para ahondar más se pueden consultar R for data science {% cite wickham2016r%}, ggplot2: elegant graphics for data analysis {% cite wickham2016ggplot2%}, y R graphics cookbook {% cite chang2018r%}. A continuación se muestra la manera en la cual se puede graficar la restricción de ejemplo $r.1=𝑥_2=4−\frac{4}{3}𝑥_1$

#1. Se carga el paquete
library(ggplot2)
#2. Se define la función relacionada con la restricción 1 (r.1)
r.1 <- function(x) 4-4/3*x
#3. Se declaran los límites:
#Límite inferior (l.i)
l.i <- 0 #Las variables normalmente son restrictas y mayores o iguales a cero
#Límite superior (l.s)
l.s <- 10 #Límite arbitrario

Una vez definida la estructura de la función, se procede a graficar utilizando el paquete ggplot2

#Se acostumbra a almacenar la gráfica en una variable denominada p (por gráfico en inglés)
#Se genera un conjunto de datos artificial para generar la estructura base
p <- ggplot(data = data.frame(x = 0), mapping = aes(x = x))
#Se agrega la función definida previamente, incluyendo los límites.
p + stat_function(fun = fun.1) + xlim(l.i,l.s)

En resumen, la ejecución del código y su respectiva gráfica se indica a continuación.

#1. Se carga el paquete
library(ggplot2)
#2. Se define la función relacionada con la restricción 1 (r.1)
r.1 <- function(x) {4-4/3*x}
#3. Se declaran los límites:
#Límite inferior (l.i)
l.i <- 0 #Las variables normalmente son restrictas y mayores o iguales a cero
#Límite superior (l.s)
l.s <- 10 #Límite arbitrario

#Se acostumbra a almacenar la gráfica en una variable denominada p (por gráfico en inglés)
#Se genera un conjunto de datos artificial para generar la estructura base
p <- ggplot(data = data.frame(x = 0), mapping = aes(x = x))
#Se agrega la función definida previamente, incluyendo los límites.
p + stat_function(fun = r.1) + xlim(l.i,l.s)

Como tal, la función resultante genera una gráfica de línea recta, donde a medida que aumenta el valor de la variable en el eje de las abscisas, disminuye el de la variable ordenada. Como por defecto el paquete de ggplot2 genera un gráfico con ejes denominados $x$ y $y$, a continuación, se agrega una estructura de código que permite personalizar el gráfico. El parámetro colour permite cambiar el color del gráfico, mientras labs configura el título de los ejes para $x_1$ labs(x = "x1"), y $x_2$ labs(y = "x2") respectivamente.

p + stat_function(fun = r.1, colour = "red")+ xlim(l.i,l.s) + labs(x = "x1") + labs(y = "x2")

De la recta se puede concluir que existen infinitos puntos $(x_1,x_2)$ que satisfacen la restricción, en consecuencia, la decisión será cuál par de puntos optimiza la función FO.

Para finalizar, se generará a modo de ejemplo otra restricción con el fin de indicar cómo graficar múltiples restricciones utilizando el código de ggplot2. Se considerará la siguiente restricción:

\begin{equation*} r2:𝑥_2=-1+𝑥_1 \end{equation*}

Repitiendo el proceso, es necesario definir la segunda función con sus respectivos colores, además. se debe definir un rango para cada una de las restricciones, por ejemplo, para la restricción 1 es: rango.1 <- data.frame(x=c(l.i, l.s), Functions = factor(1)), donde x=c(l.i, l.s) indica el dominio de la variable $x_1$, y Functions = factor(1)) permite agrupar múltiples funciones como un factor. En cuanto a los colores, la paleta se asigna manualmente mediante el código mostrado anteriormente scale_colour_manual(values = c("#1B9E77", "#D95F02") (los dos primeros colores). Para finalizar, labels = c("Restricción 1", "Restricción 2")y labs(x = "x1") + labs(y = "x2") generan las leyendas y títulos de los ejes respectivamente.

#Primera restricción
r.1 <- function(x) 4-4/3*x
#Segunda restricción
r.2 <- function(x) -1+x
#Rangos para cada restricción
rango.1 <- data.frame(x=c(l.i, l.s), Functions = factor(1))
rango.2 <- data.frame(x=c(l.i, l.s), Functions = factor(2))
#Generación del gráfico
ggplot(NULL, aes(x=x, colour=Functions)) +
  stat_function(data = plot.range1, fun = r.1) +
  stat_function(data = plot.range2, fun = r.2) +
  scale_colour_manual(values = c("#1B9E77", "#D95F02"), labels = c("Restricción 1", "Restricción 2")) +
  labs(x = "x1") + labs(y = "x2")

Es importante resaltar que, cuando se trabajan con restricciones de igualdad es posible que no exista una solución, para el caso mostrado anteriormente, existe una única solución y corresponde con el punto donde ambas soluciones se cruzan, es decir, el punto que cumple con ambas restricciones.
